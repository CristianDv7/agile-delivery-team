# Requisitos Candidatos — Ubicate

## Funcionales

- **[R-01]** El sistema debe permitir a un comerciante registrar su local con nombre, tipo de negocio, ubicación (ingreso manual o GPS), horario de atención y lista de servicios o productos.
  - Tipo: funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_02_cafeteria.md, entrevista_03_papeleria.md, entrevista_04_comida_rapida.md, entrevista_05_peluqueria.md · Comerciante Local

- **[R-02]** El sistema debe mostrar un perfil público del local con: horario, ubicación en mapa, servicios o productos, fotos, métodos de pago y enlace de contacto directo.
  - Tipo: funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_02_cafeteria.md, entrevista_03_papeleria.md, entrevista_04_comida_rapida.md, entrevista_05_peluqueria.md · Comerciante Local / Cliente Consumidor

- **[R-03]** El sistema debe permitir buscar locales por proximidad geográfica al usuario, ordenando los resultados de más cercano a más lejano.
  - Tipo: funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_03_papeleria.md, entrevista_04_comida_rapida.md, entrevista_05_peluqueria.md · Cliente Consumidor

- **[R-04]** El sistema debe permitir buscar locales por tipo de servicio o producto (p. ej. "imprimir", "corte de pelo", "recargas", "comida rápida").
  - Tipo: funcional
  - Origen: entrevista_03_papeleria.md, entrevista_05_peluqueria.md · Cliente Consumidor

- **[R-05]** El sistema debe incluir un botón de contacto directo por WhatsApp en el perfil del local, accesible sin que el consumidor necesite conocer el número de antemano.
  - Tipo: funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_04_comida_rapida.md, entrevista_05_peluqueria.md · Comerciante Local / Cliente Consumidor

- **[R-06]** El sistema debe permitir al comerciante publicar y actualizar promociones, combos o precios destacados desde el perfil del local.
  - Tipo: funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_02_cafeteria.md, entrevista_04_comida_rapida.md · Comerciante Local

- **[R-07]** El sistema debe permitir al comerciante subir fotos del local, del menú o de trabajos realizados (portafolio).
  - Tipo: funcional
  - Origen: entrevista_02_cafeteria.md, entrevista_04_comida_rapida.md, entrevista_05_peluqueria.md · Comerciante Local

- **[R-08]** El sistema debe indicar en el perfil si el local realiza delivery y los métodos de pago que acepta (efectivo, transferencia, tarjeta).
  - Tipo: funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_04_comida_rapida.md · Comerciante Local / Cliente Consumidor

## No funcionales

- **[R-09]** El registro inicial del local debe poder completarse en menos de 5 minutos, sin formularios de más de una pantalla a la vez.
  - Tipo: no funcional
  - Origen: entrevista_03_papeleria.md · Comerciante Local

- **[R-10]** Actualizar una promoción o dato del perfil (horario, precio, foto) debe tomar menos de 2 minutos desde que el comerciante abre la aplicación.
  - Tipo: no funcional
  - Origen: entrevista_02_cafeteria.md, entrevista_04_comida_rapida.md · Comerciante Local

- **[R-11]** El perfil del local debe permanecer visible y accesible para los consumidores aunque el comerciante no lo actualice diariamente.
  - Tipo: no funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_03_papeleria.md · Comerciante Local

- **[R-12]** El sistema debe ser usable desde smartphone, dado que los comerciantes operan principalmente desde el teléfono (WhatsApp es su único canal de contacto actual).
  - Tipo: no funcional
  - Origen: entrevista_01_tienda_barrio.md, entrevista_04_comida_rapida.md · Comerciante Local
